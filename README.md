# FARMS International Website

Source code for [farmsinternational.com](https://farmsinternational.com) — a PHP website hosted on DreamHost shared hosting.

## Stack

- **HTML/CSS/JS** — Bootstrap 4, jQuery, Font Awesome
- **PHP** — PHPMailer for contact/mailing-list form submissions, reCAPTCHA v2 (invisible)
- **Build** — esbuild via `build.mjs` (bundles + minifies JS and CSS into `src/dist/`)
- **Tests** — Playwright (Chromium + Firefox) against a local `http-server`
- **CI/CD** — GitHub Actions: build, test, deploy via lftp SFTP on push to `master`
- **Analytics** — Google Analytics 4 (`G-HE9ZHJ520Q`) in `<head>`, event tracking via `src/js/analytics.js`

## Key features

### Interactive world map

The "Where We Work" section renders an SVG world map with highlighted countries grouped into four regions. The map is defined in `src/js/world-map.js`.

To add or remove highlighted countries, edit the `regions` array at the top of `world-map.js`. Each region has a `name`, an array of ISO 3166-1 alpha-2 `codes`, and label coordinates (`labelX`, `labelY`). The SVG source (`src/img/world-map.svg`) was generated from Natural Earth 110m data and contains paths for 177 countries identified by alpha-2 code.

Features:
- Hover tooltips showing country names
- Smooth highlight transitions
- Responsive across mobile/tablet/desktop
- `<noscript>` fallback to a static PNG

### Analytics event tracking

Events are tracked via `data-track` attributes — no inline `onclick` handlers. The gtag snippet is in `<head>` on all pages (per Google's recommendation), ensuring `gtag()` is defined before the bundle executes.

| Attribute | Value | GA4 Event | Key Parameters |
|---|---|---|---|
| `data-track` | `donation` | `donation_click` | `link_text`, `link_url` |
| `data-track` | `pdf` | `file_download` | `file_name`, `file_extension`, `link_url` |
| `data-track` | `outbound` | `outbound_click` | `link_url`, `link_domain`, `outbound` |
| `data-track` | `newsletter-signup` | `sign_up` | `method` |
| `data-track-label` | any string | — | overrides default label |

Video play events (`video_play`) are tracked automatically via the Vimeo Player API and YouTube iframe API — not via click handlers on wrapper divs (which cannot capture cross-origin iframe interactions).

Form submissions fire `form_submit` (with `form_name`) from `farms.js` on successful AJAX response via `window.FARMS.fireEvent()`.

GA4 Enhanced Measurement handles generic outbound link tracking automatically. The explicit `data-track="outbound"` is for curated links where you want a custom `outbound_click` event with specific labels.

### `brand-guide.html`

Developer-only reference for the site's color palette, typography, and UI components. Excluded from CI/CD deployment and does not include Google Analytics. Do not link to it from any live page.

## Local development

**Prerequisites:** Node.js 24+, PHP (for form handling on the server)

```bash
npm install
npm run build      # bundle JS + CSS -> src/dist/
npm run dev        # watch mode (rebuilds on change)
npm test           # run Playwright tests (headless)
npm run test:ui    # run Playwright tests with UI
```

Tests automatically start `http-server` on `src/` at port 8080.

## Deployment

Deployment runs automatically via GitHub Actions on every push to `master`, gated behind passing tests.

**First-time setup:**
1. Copy `config.example.php` to `src/config.php` and fill in credentials
2. Set these GitHub repository secrets:
   - `DEPLOY_HOST` — SFTP hostname (e.g. `farmsinternational.com`)
   - `DEPLOY_USER` — SFTP username
   - `DEPLOY_PASSWORD` — SFTP password
   - `DEPLOY_PATH` — Remote web root path (e.g. `/home/user/farmsinternational.com/`)

The deploy step uses `lftp` to mirror `src/` to the server via SFTP, excluding `brand-guide.html`, config files, and other dev-only artifacts.
