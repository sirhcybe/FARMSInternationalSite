# FARMS International Website

Source code for [farmsinternational.com](https://farmsinternational.com) — a PHP website hosted on DreamHost shared hosting.

## Stack

- **HTML/CSS/JS** — Bootstrap 4, jQuery, Font Awesome
- **PHP** — PHPMailer for contact/mailing-list form submissions, reCAPTCHA v2 (invisible)
- **Build** — esbuild via `build.mjs` (bundles + minifies JS and CSS into `src/dist/`)
- **Tests** — Playwright (Chromium + Firefox) against a local `http-server`
- **CI/CD** — GitHub Actions: build → test → deploy via lftp SFTP on push to `master`
- **Analytics** — Google Analytics 4 (`G-HE9ZHJ520Q`), centralized via `src/js/analytics.js`

## Developer notes

### `brand-guide.html`

This file is a **developer-only reference** for the site's color palette, typography, and UI components. It is excluded from CI/CD deployment and does not include Google Analytics. Do not link to it from any live page.

### Analytics event tracking

Events are tracked via data attributes — no inline `onclick` handlers. Add these attributes to any element:

| Attribute | Values | Fires |
|---|---|---|
| `data-track` | `donation` | `donation_click` |
| `data-track` | `pdf` | `file_download` |
| `data-track` | `outbound` | `click` (Outbound Link) |
| `data-track` | `video` | `video_play` |
| `data-track` | `newsletter-signup` | `newsletter_signup` |
| `data-track-label` | any string | overrides default label |

Outbound links (`href` pointing off-site) are also auto-detected and tracked even without a `data-track` attribute.

Form submissions (contact form and mailing list) fire `form_submit` events from `farms.js` on successful AJAX response via `window.FARMS.fireEvent()`.

## Local development

**Prerequisites:** Node.js 24+, PHP (for form handling)

```bash
npm install
npm run build      # bundle JS + CSS → src/dist/
npm run dev        # watch mode (rebuilds on change)
npm test           # run Playwright tests (headless)
npm run test:ui    # run Playwright tests with UI
```

Tests spin up `http-server` on `src/` at port 8080 automatically.

## Deployment

Deployment runs automatically via GitHub Actions on every push to `master`, gated behind passing tests.

**Manual deploy prerequisites (first time):**
1. Copy `config.example.php` to `src/config.php` and fill in credentials
2. Set the following GitHub repository secrets:
   - `FTP_SERVER` — SFTP hostname (e.g. `farmsinternational.com`)
   - `FTP_USERNAME` — SFTP username
   - `FTP_PASSWORD` — SFTP password
   - `FTP_SERVER_DIR` — Remote web root path (e.g. `/home/user/farmsinternational.com/`)

The deploy step uses `lftp` to mirror `src/` to the server, excluding `brand-guide.html`, `dist/` source maps, and other dev-only files.
